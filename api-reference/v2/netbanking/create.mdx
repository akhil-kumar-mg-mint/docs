## Overview

Creates a net banking payment order via S2S. The response contains an `acs_template` (Base64-encoded HTML) that must be decoded and rendered in the customer's browser to redirect them to their bank's net banking portal.

This endpoint is identical to [Create Order](/api-reference/v2/order/create) but with `collection_mode: "s2s"` and `mop_type: "net_banking"`.

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| `X-Client-ID` | Your Client ID | Yes |
| `X-Client-Secret` | Your Client Secret | Yes |
| `X-Merchant-ID` | Merchant ID (PSP only) | Conditional |
| `X-API-Version` | `2.0.0` | Yes |
| `Content-Type` | `application/json` | Yes |

### Body Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `amount` | string | Yes | Payment amount (e.g. `"1000.00"`) |
| `currency` | string | No | ISO 4217 currency code. Defaults to `"INR"` |
| `reference_id` | string | Yes | Unique merchant reference ID (alphanumeric, max 50 chars) |
| `return_url` | string | Yes | URL to redirect customer after bank authentication |
| `collection_mode` | string | Yes | Must be `"s2s"` |
| `mop_type` | string | Yes | Must be `"net_banking"` |
| `netbanking_details.bank_name` | string | Yes | Name of the bank â€” must exactly match a name from the [Supported Banks](/integration-guide/v2/web-integration/s2s-netbanking#supported-banks) list |
| `buyer.name` | string | Yes | Full name of the customer |
| `buyer.email` | string | Yes | Customer email address |
| `buyer.phone` | string | Yes | Customer phone number with country code (e.g. `"+919876543210"`) |
| `buyer.address.line_1` | string | Yes | Street address |
| `buyer.address.city` | string | Yes | City |
| `buyer.address.state` | string | Yes | State |
| `buyer.address.postal_code` | string | Yes | PIN / postal code |
| `buyer.ip_address` | string | No | Customer's IP address â€” recommended for fraud prevention |
| `buyer.user_agent` | string | No | Customer's browser user-agent â€” recommended for fraud prevention |
| `product.name` | string | Yes | Product or service name |
| `product.type_of_goods` | string | Yes | `physical_goods`, `digital_goods`, or `service` |
| `invoice.number` | string | Yes | Unique invoice number |

## Response

### Success Response

```json
{
  "success": true,
  "message": "S2S Net Banking Payment Request created successfully",
  "data": {
    "order_id": "OD9123456789",
    "acs_template": "PGh0bWw+PGJvZHk+PGZvcm0gbmFtZT0icGF5bWVudF9wb3N0..."
  }
}
```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `success` | boolean | `true` if the order was created successfully |
| `data.order_id` | string | Unique order ID â€” use this to check payment status |
| `data.acs_template` | string | Base64-encoded HTML â€” decode and render in the customer's browser to initiate the bank redirect |

### Error Response

```json
{
  "success": false,
  "message": "Validation error",
  "errors": {
    "bank_code": ["This field is required."]
  }
}
```

## Next Steps

After receiving the response:

1. Decode `acs_template` using `atob()` and render the HTML in the customer's browser
2. The form auto-submits and redirects the customer to their bank's net banking portal
3. After the customer authorises payment, they are redirected to your `return_url`
4. Call [Get Order Status](/api-reference/v2/order/get-status) to confirm the payment outcome

See the full integration walkthrough in the [S2S Net Banking Integration Guide](/integration-guide/v2/web-integration/s2s-netbanking).
